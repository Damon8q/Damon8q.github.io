VMware 虚拟机Linux空间拓展
发布于 2019-03-25 / 更新于 2019-03-25 / / 热度 0 / 喜欢 0

声明 ：本过程仅本人 VMware 虚拟机Linux系统 上运行成功，其他系统上不能保证，若出现数据丢失，概不负责。

关闭虚拟机，进入设置页面，选择硬盘，增加到想要拓展的大小
打开虚拟机。root登陆
终端输入：
fdisk -l
“查看分区目录”
继续键入：
n
“ 命令 n 用于添加新分区 “
继续键入：
p
“命令 n 用于添加新分区 “
继续键入：
3
“fdisk 会让你选择主分区的编号，如果已经有了主分区 sda1 ， sda2 ，那么编号就选3 ，即要创建的该分区为 sda3
fdisk 又会让你选择该分区的开始值这个就是分区的 Start 值（ start cylinder ）；这里最好直接按回车，如果您输入了一个非默认的数字，可能会造成空间浪费”
等待完成，继续键入：
t
“改变分区系统id”
继续键入：
3
“指定分区号”
继续键入：
8e
“指定要改成的id号，8e代表LVM”
继续键入：w
“保存”
终端输入：
fdisk -l
“查看新的分区大小以及支持格式”
终端输入：
mkfs -t ext3 /dev/sda3
“格式化分区目录”
终端输入：
lvs
pvcreate /dev/sda3
vgextend centos /dev/sda3
“其中 centos 是需要扩充的目录组 可以通过df -h 查看 例如我的 /dev/mapper/centos-root centos 为组 ”
终端输入：
vgdisplay
“主要查看 Free PE / Size 8960 / 35.00 GiB，说明我们最多可以有35.00 G的扩充空间”
终端输入：
lvextend -L +34.7G /dev/centos/root /dev/sda3
“扩充我们想要增加的大小，做好稍微余留一些 避免出现分区不足的错误“
终端输入：
vim /etc/fstab
“查看拓展组中 逻辑卷 的文件系统“
e2fsck -f /dev/centos/root
出现”/dev/mapper/centos-root is mounted. e2fsck: 无法继续, 中止.” 继续进行

    如果是 逻辑卷文件系统 xfs:
    终端输入：
            xfs_growfs  /dev/centos/root 
    如果是 逻辑卷文件系统 ext3:
    终端输入：
            resize2fs /dev/centos/root 
            输出如下 “
            meta-data=/dev/mapper/centos-root isize=512    agcount=4, agsize=2424320 blks
                    =                       sectsz=512   attr=2, projid32bit=1
                    =                       crc=1        finobt=0 spinodes=0
            data     =                       bsize=4096   blocks=9697280, imaxpct=25
                    =                       sunit=0      swidth=0 blks
            naming   =version 2              bsize=4096   ascii-ci=0 ftype=1
            log      =internal               bsize=4096   blocks=4735, version=2
                    =                       sectsz=512   sunit=0 blks, lazy-count=1
            realtime =none                   extsz=4096   blocks=0, rtextents=0
            data blocks changed from 9697280 to 18741248
            ”
    终端输入：
            df -h 
            “查看内存情况”

##ok！
